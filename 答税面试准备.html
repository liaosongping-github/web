一、公司与产品全景

一句话：答税做的是「帮财务/税务人员少加班、少出错」的工具 + 「帮人搞懂政策」的内容/AI 平台。

财税 = 财务 + 税务。财务管「账怎么记、钱怎么算」；税务管「该交什么税、怎么向国家申报」。答税两边都碰，但核心更偏「税」相关软件。

1.1 五条产品线（先记故事，再记名字）

立马审 · 智汇算专业版

帮事务所快速做完「税审」

主要客户：税务师事务所、会计师事务所里的审计/税审人员

税审 ≈ 专业机构帮企业检查「税交得对不对」，并出具正式报告（像体检报告）。企业有时需要这份报告去办某些业务或自查风险。
  • 把企业账套数据导进来（对接很多财务软件）
  • 自动抽查凭证、生成工作底稿
  • 自动算常见「纳税调整」
  • 生成报告，并尽量一键去税局/税协备案
  • 还给事务所看项目进度、回款等经营数据
立马报 · 智汇算企业版

帮企业（或代账）批量把税报出去

主要客户：企业财务、代理记账公司、有很多分子公司的集团

报税 / 纳税申报 ≈ 按规定时间，把「这个月/这个季度该交多少税」算清楚，填表提交给电子税务局（国家收税的网上大厅）。
  • 从财务软件采数 → 自动填表、校验
  • 多税种一起管，减少漏报
  • 多主体（很多公司）可批量申报，少反复登录各地税局
  • 用自动化（含 RPA）减轻人工点鼠标
立马答 · TaxMind

财税版「专业 ChatGPT」

主要客户：会计、税务顾问、企业财务、需要查政策的人

大模型 / AI 问答 ≈ 你用大白话问「研发费用能不能加计扣除」，它结合法规库尽量给出合规、可引用的回答。不是替代专家签字，而是先快速答疑。
  • 绑定答税自建法规库（政策、问答、文章很多）
  • 支持联网追新政策、语音提问、历史对话
  • 常作为会员权益里的「不限次 AI 咨询」
立马查 · 云鹏报告

给企业做「税务+经营风险体检报告」

主要客户：银行等金融机构（放贷风控）、财税机构、企业老板

企业画像 / 风险报告 ≈ 把工商信息、纳税情况、发票异常、欠税处罚、上下游交易等拼成一份「这家公司靠不靠谱」的说明，方便决策。
  • 看税负是否异常、有没有欠税/处罚
  • 看发票红冲作废是否可疑
  • 看供应链集中度、财务健康指标
立马记 · 智能记账

让每月做账更快

主要客户：代理记账公司、中小企业财务

记账 / 凭证 ≈ 把每一笔生意（买东西、卖东西、发工资）记成会计能看懂的「流水账条目」。凭证记对了，后面才报得了税、出得了报表。
  • 取票后尽量自动生成凭证
  • 自动结转、出报表
  • 电子账本，少纸质、少丢数据
平台层 · 获客与知识服务

法规库 / 问答 / 云课堂 / 专家咨询

个人会员 + 机构用户；也是软件产品的「流量入口」

  • 查政策、看课、问专家 → 建立信任
  • 再转化成会员，或升级买智汇算等付费工具
  • 官网还提到「智联平台」（审计资料闭环管理），可作延伸了解

1.2 税审产品工作流（面试可画）

采数导账→ 智能抽凭→ 生成底稿→ 纳税调整→ 出报告→ 备案归档
底稿 ≈ 审计/税审过程中留下的「检查记录表」。抽凭 ≈ 从海量记账凭证里按规则挑出要重点检查的那些。纳税调整 ≈ 会计利润和税法认可的利润不一致时，按税法规则改过来再算税。

二、商业模式（谁付钱、怎么付钱）

答税不是只卖一个 App，而是「内容/AI 引流 → 会员订阅 → 专业工具年费/按项目 → 大客户私有化」的组合拳。

2.1 两条客群线

工具线（ToB 为主）

  • 事务所买「立马审」提效接更多项目
  • 企业/代账买「立马报」「立马记」少加班少出错
  • 金融机构/机构买「云鹏报告」做风控或诊断

知识线（个人 + 从业者）

  • VIP / 课程 / 黄金会员：课 + 专家咨询次数 + TaxMind
  • 先免费搜法规、试用 AI，再付费升级

2.2 收入形态（官网公开卖法）

怎么收费 人话理解 对应产品
个人会员年费 像「得到/樊登」式会员：课、咨询次数、AI 问答打包卖 VIP / 课程会员 / 黄金会员 + TaxMind
软件按年订阅（SaaS） 每年交费，云端登录使用，不用自己装服务器 智汇算企业版等
按项目计价 按税审项目量买额度（官网写专业版约 100 项目起购） 智汇算专业版（立马审）
私有化部署 软件装到客户自己机房/云上，数据不出客户环境，通常更贵、要人跟进 独享版
报告试用转付费 先看一份风险报告值不值，再持续买 云鹏报告
SaaS ≈ Software as a Service，软件当服务卖：你交年费，打开网页/客户端就能用，厂商负责升级维护。私有化 ≈ 软件「搬进」客户自己的环境，更强调安全与定制。

2.3 交叉销售（产品怎么互相带量）

  • 法规库 / 课堂 / TaxMind 拉新 → 转化会员
  • 会员里的代账/财务人员 → 升级买 立马记 + 立马报
  • 事务所客户用 立马审 做项目 → 可能延伸咨询、培训、报告类服务
  • 云鹏报告 面向银行/风控，是另一条「数据服务」收入线，不完全依赖会员

2.4 对「财税产品经理」意味着什么

你日常不是做商品详情页，而是：把复杂专业流程拆成功能模块、保证合规不出错、对接财务软件和税局、 用 AI 缩短作业时间、并想清楚「免费功能」和「付费转化」怎么配。 这和你做过的 B 端后台、中台、结算、SaaS 封装、AI 提效 是同一类产品思维。

五、差距与 7 天补齐路径

5.1 主要差距（白话）

  • 还不太熟：企业每个月/季/年要报哪些常见税，电子税务局大概怎么操作
  • 还不太熟：事务所做企业所得税汇算鉴证时，从导账到出报告的一天长什么样
  • 还没建立：答税五个产品各自「谁付钱、怕什么、爱什么功能」的用户画像
  • 竞品只知皮毛：同赛道还有哪些报税/税审/代账工具（知道名字和差异即可）
汇算清缴 ≈ 企业所得税不是只按月估着交，年底还要按全年真实利润「算总账」，多退少补。很多税审项目就围绕这件事。

5.2 七天速补清单

D1
搞清答税五产品：打开官网每个产品页，用自己的话各写 3 句「帮谁、解决啥、怎么收费」。试用 TaxMind / 云鹏（若可注册）。
D2
税种扫盲：只记增值税、企业所得税、个人所得税、附加税费、印花税「各自对普通人意味着什么」+ 大概多久报一次。不求背法条。
D3
跟一遍报税故事：想象「一家有 10 个子公司的集团财务」月底痛点；对照立马报的卖点写「如果我是 PM，第一期做哪 3 个功能」。
D4
跟一遍税审故事：想象「税审助理外勤导账半天」；对照立马审流程，画出你自己的流程图（可用任意工具）。
D5
业财迁移故事打磨：把「统一结算中心」改写成「账-票-款-（税）」叙事，录音 3 分钟自我介绍练到顺。
D6
AI 故事对齐 TaxMind：用你的「真相库工作流」类比「法规库 + 作业流嵌入」；准备 1 个「AI 幻觉/合规风险」怎么控的回答。
D7
模拟面试：对着本文 6.2 / 6.4「追问」自问自答；查 2～3 个竞品官网首页差异（功能定位一句话即可）。

边界：AI 提建议，专家/规则负责合规签发，防幻觉。
为什么从电商转到财税？ 不是追风口，是能力延伸：交易 → 结算 → 合规 → 税；
想在专业垂直领域做深；认同答税「工具 + 知识 + AI」。

产品经理常规题

可能问什么建议怎么答
负责立马报或立马审，前 90 天做什么？ 30 天：跟通一条完整用户路径，分清审/报，摸清耗时、申报成功率、续费等指标。
60 天：接一类迭代（稳定/提效/转化）。
90 天：对负责模块提出可量化目标并上线验证。
主路径先选一条，别两条都深挖。
财务软件已能报税，立马报会不会多余？ 单主体、单软件往往够用；
差异在：代账/多主体批量、多财务软件采数、申报期规则与对接。
和金蝶用友是部分重叠、部分互补。
税审报告和云鹏报告有什么区别？ 税审报告：事务所专业鉴证，用于汇算/办事/自查。
云鹏报告：风险画像，用于信贷风控/诊断。
一个鉴证，一个情报。
怎么做需求分析？如何与专家协作？ 你负责：结构化、优先级、流程抽象；
专家负责：规则正确性；
规则做成可配置，避免专家绑在每个项目上。
用过 Axure 吗？ 如实说工具栈；强调能画清流程和原型；
可现场画立马审主路径证明结构能力。

压力 / 细节题（有备无患）

可能问什么回答方向
薪资预期？ 面议；先谈匹配与可贡献，再谈区间。
稳定性 / 为何看机会？ 正向：想深耕专业垂直；避免抱怨前司。
记账、报税、税审顺序？ 企业日常：记账 → 按期报税；
税审是外部体检，非每单必经。
采购/销售单财务节点？ 货票款三单匹配；
销售侧重交付确认收入；
采购侧重收货入库 + 收票抵扣。
竞品谁？我们第几？ 按赛道拆；不说全国第几；
税审垂直有认知，报税偏红海挑战者。
考前优先背熟：结算中心故事、南网对接故事、审 vs 报、两份报告区别、立马报是否多余。被问税务细节时：不会就说不会 + 说明怎么向专家求证，比硬编强。

6.3 三个可讲项目（STAR 提纲）

故事 A · 统一结算中心（对标：账票款 / 合规 / 中台）

S 多业务线结算分散、合规风险高，财务入账慢。
T 从 0 到 1 建统一结算枢纽。
A 归集订单→账单清洗转换→推财务入账；对接航信自动开票；多部门 4 个月打通。
R 统一账票款管理，减少重复建设，沉淀数据底座。
迁移句：这与报税/税审前的「数据归集—规则处理—对外报送」是同一类结构。

故事 B · 企业商城 SaaS 化（对标：ToB SaaS / 可配置）

S 内部集采能力要对外输出。
T 封装核心能力做可配置采购平台。
A 对接头部供应商与 OA/费控；支持客户个性化。
R 外部客户额外创收 1500 万+。
迁移句：理解多客户差异化配置，类似不同事务所/企业的税审与申报差异。

故事 C · AI + 产品真相库（对标：TaxMind / 智能作业）

S 需求评估依赖人口径，和代码现状脱节。
T 用 Agent 把 PRD 与代码双源建成「真相库」工作流。
A 初始化→影响分析→PRD→上线→回写知识库。
R 缩短评估与方案起草周期。
迁移句：和「法规库 + 大模型 + 嵌入税审/咨询作业」是同一产品哲学。

6.4 可能追问与答法要点

可能问什么怎么答（要点）
你不懂税务怎么办？ 承认缺口；展示相邻经验；说明已试用/梳理产品与 7 天计划；强调 PM 价值是抽象流程与落地，专家规则可共建。
立马审和立马报区别？ 审=事务所做检查出报告；报=企业/代账向税局申报交税。客群与作业流不同。
你怎么看 TaxMind？ 垂直知识 + 作业场景嵌入才有壁垒；要解决幻觉与责任边界（AI 建议 vs 专家签发）。
如何做竞品分析？ 按客群（事务所/企业/代账）、关键作业步骤、对接能力、定价、AI 深度五维对比，不只比功能清单。
0→1 你怎么保证质量？ 用结算中心/南网案例：范围冻结、联调清单、灰度、业务陪跑、指标验收。
为何从电商转到财税？ 不是追风口，是能力自然延伸：交易→结算→合规→税；想在更专业垂直领域做深。
薪资 18–30K 预期？ 按简历「面议」准备区间；强调匹配度与可落地贡献，避免第一轮死咬数字。
用过 Axure 吗？ 如实说工具栈；强调流程图/原型能力可迁移；可现场画立马审主流程证明结构能力。
如果让你做 Q3 规划？ 先访谈 3 类用户；选一条主路径（如多主体申报或税审采数）打磨；AI 做辅助提效；用留存/耗时/差错率衡量。
如何与专家型同事协作？ 你负责结构化与优先级；专家负责规则正确性；用「规则配置化」减少把专家绑在每个项目上。
